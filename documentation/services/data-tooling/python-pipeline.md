# Python CSV Pipeline

Optional data tooling to convert FIFA World Cup CSV datasets into sweepstake-ready JSON. **Not part of the Next.js runtime** — the app never imports Python code.

## Current role in the project

The **active 2026 bootstrap** does not depend on this pipeline:

- Teams and tournament metadata are hand-authored in `data/seeds/wc_2026/`
- Fixtures are generated by `scripts/wc2026/gen-fixtures.ts` (TypeScript manifest) and committed as `data/seeds/wc_2026/fixtures.json`
- Upload to Supabase is `npm run wc2026:upload`

The Python project remains for **future CSV ingestion** (e.g. when FIFA publishes official 2026 schedules) and for **historical analysis** of the bundled Kaggle dataset.

## Project layout

```
pyproject.toml                          uv project root (repo root)
data/raw/
  FIFA World Cup 1930-2022 All Match Dataset.csv   Kaggle source
  team_names.py                         EN → ES name map + aliases
  utils.py                              normalization helpers (pandas)
```

There is **no** CSV-to-JSON converter script in the repo right now — only shared helpers (`utils.py`, `team_names.py`) and the Kaggle CSV. Fixtures for 2026 are produced with `scripts/wc2026/gen-fixtures.ts`, not Python.

2022-only notebooks and strip scripts were removed with the wc2022 retirement.

## `team_names.py`

`TEAM_NAME_ES` dict maps English CSV names (and common aliases) to Spanish display names used in the app and seed JSON, e.g. `"Netherlands" → "Países Bajos"`, `"Korea Republic" → "Corea del Sur"`.

Extend this file when adding teams from a new CSV source. The 2026 seed teams were authored directly in `teams.json`, not exported from this map.

## `utils.py` — shared transforms

Library functions (importable from notebooks or a future converter):

| Function | Role |
|----------|------|
| `normalize_col_name` / `build_column_map` / `first_existing` | Tolerant CSV column matching |
| `translate_team` | Apply `TEAM_NAME_ES`; strip Kaggle HTML artifacts |
| `normalize_stage` | CSV stage string → app `fase` (`fase_grupos`, `dieciseisavos`, `octavos`, …) |
| `normalize_group` | `"Group A"` → `"A"` |
| `detect_extra_time_and_penalties` | Infer `prorroga` / `penaltis` from win-condition text |
| `infer_winner` | Derive `ganador` from flags or scoreline; group draws → `"empate"` |
| `load_partidos_index` | Index JSON grouped by jornada key `(fase, equipo_1, equipo_2)` |

Output `fase` values align with `FASE_VALUES` in `src/lib/fixtures/pythonFormat.ts`.

## Target JSON shape

When building or converting matches, emit objects matching `PythonMatchSchema`:

```json
{
  "external_id": "wc2026_md1_a_mex_rsa",
  "fase": "fase_grupos",
  "tipo_partido": "grupo",
  "jornada": 1,
  "grupo": "A",
  "equipo_1": "México",
  "equipo_2": "Sudáfrica",
  "fecha": "2026-05-29 18:00:00"
}
```

Rules:

- `fecha` is **Europe/Madrid wall time** without timezone suffix (upload converts via `madridLocalToUtcIso`)
- `external_id` must stay stable across re-uploads (upsert key with `tournament_id`)
- Knockouts without known teams: `"TBD"` on both sides (stored as placeholders in DB)
- Result fields (`marcador_*`, `prorroga`, `penaltis`, `ganador`) are optional in seed JSON; upload ignores them

Validate with the same Zod schema the app uses: `PythonMatchSchema` in `src/lib/fixtures/pythonFormat.ts`.

## Legacy files under `data/partidos/`

Only `data/partidos/README.md` remains — a pointer to the canonical seed path. Older grouped JSON extracts were removed.

## Known historical issues (2022 pipeline)

The retired 2022 workflow extracted dates from Kaggle CSV where **every match had `02:00:00`** as a placeholder hour — not real kickoff times. That motivated:

1. Hand-fixing dates in admin UI, then
2. Moving 2026 generation to `gen-fixtures.ts` with explicit Madrid 18:00 slots

Any new CSV converter should parse real kickoff datetime from the official source, not copy the Kaggle placeholder pattern.

## Workflow: CSV → seed JSON (when converter exists)

1. Place or update CSV under `data/raw/`
2. Run converter (future `uv run convert-world-cup-csv` or notebook) → flat JSON array
3. Validate against `PythonMatchSchema` (manual or script)
4. Write to `data/seeds/wc_2026/fixtures.json` (or merge team changes into `teams.json`)
5. `npm run wc2026:upload` locally; verify counts
6. Commit JSON + run prod upload with `--confirm-prod` when ready

For small fixture edits (knockout bracket), prefer admin JSON import — `documentation/implementations/admin-fixtures-json-import.md`.
